# Мастер-класс по рефакторингу бэковых тестов при помощи ИИ-агентов

На данном мастер-классе мы будем исследовать возможности ИИ-агентов для рефакторинга бэковых автотестов на примере веб-калькулятора с REST API. Будем делегировать нейросетям рутину: исправление техдолга, генерацию сложных структур и массовый рефакторинг — всё это на примере плагина Roo Code для прямого взаимодействия с кодом в IDE. Мы разберем два ключевых подхода: итеративное выполнение мелких задач и планирование реализации под контролем оркестратора. Все задания будут выложены в этот репозиторий после мастер-класса.

## Подготовка и настройка окружения

### 1. Настройка VSCode и RooCode

Подробные инструкции по установке и настройке VSCode, RooCode и провайдера моделей на примере Cerebras AI см. в [README прошлой версии мастер-класса](https://github.com/fortrest-jr/sqa37-calc-task#подготовка-и-настройка-окружения).

**Важно**: Сейчас Cerebras AI не поддерживается в RooCode поэтому нужно подобрать альтернативный провайдер моделей. Для повседневного использования рекомендую Sourcecraft

### 2. Получение кода проекта

1. Проверьте установлен ли Git, набрав в консоли:
    ```bash
    git --version
    ```

    Если нет — [установите git](https://git-scm.com/downloads).

2. Клонируйте репозиторий с проектом:

    ```bash
    git clone https://github.com/fortrest-jr/sqa38-calc-task2.git
    cd sqa38-calc-task2
    ```


### 3. Настройка окружения проекта
Выполняется после клонирования репозитория, в созданной папке.

1. Убедитесь, что у вас установлен uv, набрав в консоли:
    ```bash
    uv --version
    ```
    Если нет, [установите uv](https://docs.astral.sh/uv/getting-started/installation/)

2. Установите зависимости:
    ```bash
    uv python install
    uv venv
    uv pip install -r requirements.txt
    ```

3. Откройте в VSCode папку проекта:
    ```bash
    code .
    ```

### Расширение возможностей агентов

Для углубленного изучения и расширения возможностей RooCode и аналогов:
* **Agents.md** — Документация по созданию и управлению агентами в любых агентах. Позволяет универсально определять поведение и стратегии ИИ-агента.
* **Skills** — Система расширения функциональности агентов. Можно создавать собственные skills для специфических задач и интегрировать их в рабочий процесс чтобы не повторяться с описанием разных возможностей в каждой новой задаче.
* **MCP (Model Context Protocol)** — Протокол для расширения контекста и интеграции внешних инструментов и сервисов с ИИ-агентами. Позволяет подключать дополнительные источники данных и функциональности.

Эти инструменты полезны для разработки собственного harness (набора инструментов и конфигураций) под большинство задач и рабочих процессов в любых агентах, включая Roo Code.
